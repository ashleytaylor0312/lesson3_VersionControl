Highlight #1 Version Control

Version control is a system that record changes that are made to a file or set group of files over time. This information can be used to reference back to specific versions later. The Version Control System (VCS) could be important for individuals who are trying to pursue a career in graphic or web design. This system allows you to revert a selected file or entire project back to the previous state, see who last modified the file, review changes made, and more. This interest me because I am definitely going to the graphic/web design field and it would beneficial to look back this information. I would say this is important for us in class to be able to look back at what changes we made and fix any errors we may have.

Highlight #2 History of Git

The development of Git began with Linux kernel, which is an open software project. Linux kernel and a commercial company that developed BitKeeper had a mutual community. In 2005, that community broke down and have the “free-of-charge” status cancelled. From there, Linux kernel decided to develop their own tool. They used their past experiences with BitKeeper to help construct their desired tool. They were looking to have features such as speed, a simple design, to be fully distributed, a strong support for parallel branches, and to be able to manage large software like Linux with speed and data. It is important to know the background of the system we are going to work with for assignments.

Highlight #3 Basics of Git

A Git is a version control system. It helps to check track of files and makes it easy for groups of people to work with the files. Git is special because unlike other systems, Git thinks of the system more as data. Each time someone commits or save a project their working on, Git will take a screenshot of the moment and store it. Of course, if nothing has changed with the file then it will just be linked to the previous “screenshot” or file that was already stored by Git. It does need operate with local files. This is important for our class because we can always go back and reevaluate any changes we make. If we have to fix any errors, we can look back at what changed and how we can improve it.

Highlight #4 Git Repository

A Git repository is where the Git stores its information. It contains commit objects and heads, which are sets of referenced to commits. There are two ways that a Git repository can be obtained. The first option is turning a local directory that is not currently under version control to a Git repository. This process does include using the command “cd” to change directories. The syntax of the common will depend on that type of system you are working on. The second option is to clone an existing Git repository from somewhere else. This is important to know because we can locate where any files that we commit are.

Highlight #5 Changes to Repository

The final reading is I found helpful was Recording Changes to Repository. The files you are working must be either a tracked or untracked state. A tracked state means that Git knows the files already. They could have been modified, unmodified, or staged. Untracked are other files that have not been modified, unmodified, or staged. In this reading, if gave details for checking the status of your file, tracking new files, staged modified files, ignoring files and more.
